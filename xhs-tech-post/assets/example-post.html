post body (正文)

把 43 个 tool 全量塞进 prompt 是项目第一个版本的做法，跑两轮 LLM 就能选错；后面拆成 7 个 skill 做一级目录、再下沉到 tool，prompt 砍了 60%、选择准确率回到能用的水平。

这套结构在 30B MoE 上稳了，70B dense 还在压 latency。skill 的边界划分本身是个工程活，划错一次返工很疼。

做过类似 multi-agent harness 的同行欢迎评论区交流。
